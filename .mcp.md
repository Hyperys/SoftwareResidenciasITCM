# MCP — Software de Residencias ITCM
> Contexto compacto del proyecto para asistentes de IA. Actualizar al cambiar arquitectura.

---

## 1. Propósito
Sistema de escritorio para gestionar las **residencias profesionales** del TECNM Campus Ciudad Madero.  
Permite registrar residentes, empresas, asesores, carreras, documentos y generar expedientes PDF.

---

## 2. Stack Tecnológico

| Capa | Tecnología | Versión / nota |
|---|---|---|
| **Escritorio** | Electron | `electron/package.json` |
| **Frontend** | React 18 + Vite | SPA, sin router, todo en `App.jsx` |
| **Backend** | Flask (Python) | Puerto `5000`, API REST JSON |
| **Base de datos** | MySQL | DB: `residencias_tecnm_madero` |
| **Estilos** | CSS-en-JS inline | Todo en `App.jsx` (template string `styles`) |
| **Empaquetado** | PyInstaller + electron-builder | Genera `app.exe` + instalador NSIS |

---

## 3. Arquitectura

```
SoftwareResidencias/
├── electron/           # Cáscara de escritorio
│   └── main.js         # Lanza Flask, abre BrowserWindow en localhost:5000
├── backend/            # API Flask
│   ├── app.py          # Registra blueprints, sirve SPA en /
│   ├── database.py     # ⚠️ NO en repo — credenciales MySQL (ver database.example.py)
│   ├── database.example.py
│   ├── requirements.txt
│   └── routes/         # Un blueprint por módulo
│       ├── residentes.py
│       ├── proyectos.py
│       ├── asesores.py
│       ├── empresas.py
│       ├── carreras.py
│       └── backup.py
├── proyecto/           # Frontend React/Vite
│   └── src/
│       ├── App.jsx     # ⚠️ Archivo monolítico (~3500 líneas): todo el UI
│       └── main.jsx    # Entry point React
├── database/
│   └── schema.sql      # DDL completo + datos de prueba
├── expedientespdf/     # ⚠️ NO en repo — archivos físicos de expedientes
└── backup/             # ⚠️ NO en repo — backups de la BD
```

### Flujo de arranque (producción)
1. Electron ejecuta `resources/backend/app.exe` (Flask compilado con PyInstaller)
2. Pollean `http://127.0.0.1:5000` hasta que responde
3. Abren `BrowserWindow` apuntando a `http://127.0.0.1:5000`
4. Flask sirve `proyecto/dist/index.html` como SPA + API en `/api/*`

### Flujo de arranque (desarrollo)
```bash
# Terminal 1: Backend
cd backend && python app.py        # requiere database.py configurado

# Terminal 2: Frontend
cd proyecto && npm run dev         # Vite en :5173

# Terminal 3: Electron (opcional)
cd electron && npm start
```

## 4. Base de Datos — Tablas clave

| Tabla | PK | Descripción |
|---|---|---|
| `residentes` | `num_control` VARCHAR(10) | Entidad principal. Estados: `activo`, `cerrado` |
| `empresas` | `id` INT | Empresas receptoras de residentes |
| `asesores` | `id` INT | Internos (docentes) y externos (empresa) |
| `carreras` | `id` INT | ISC, II, etc. |
| `especialidades` | `id` INT | Subdivisión de carrera |
| `documentos` | `id` INT | Archivos PDF asociados a un residente |
| `configuracion` | `clave` VARCHAR | Rutas de almacenamiento |
| `backups_log` | `id` INT | Historial de backups |

#### Columnas críticas de `residentes`
```sql
es_expediente   TINYINT(1)  -- 0=proyecto en curso, 1=expediente formal
estado_hito1    ENUM(pendiente, en_progreso, completado)
estado_hito2    ENUM(pendiente, en_progreso, completado)
estado_hito3    ENUM(pendiente, en_progreso, completado)
estado          ENUM(activo, cerrado)
ruta_archivos   VARCHAR     -- ruta relativa dentro de expedientespdf/
```

## 5. API — Endpoints

**Base URL**: `http://127.0.0.1:5000/api`

| Método | Ruta | Acción |
|---|---|---|
| GET | `/residentes` | Lista (filtros: `search`, `anio`, `semestre`, `estado`) |
| GET | `/residentes/:num_control` | Detalle + documentos |
| POST | `/residentes` | Crear residente + carpeta física |
| PUT | `/residentes/:num_control` | Editar residente |
| DELETE | `/residentes/:num_control` | Eliminar (param: `?eliminar_archivos=true`) |
| POST | `/residentes/:num_control/documentos` | Subir PDF (multipart) |
| DELETE | `/documentos/:doc_id` | Eliminar documento |
| GET/POST/PUT/DELETE | `/proyectos` | CRUD proyectos (módulo análogo) |
| GET/POST/PUT/DELETE | `/asesores` | CRUD asesores |
| GET/POST/PUT/DELETE | `/empresas` | CRUD empresas |
| GET/POST/PUT/DELETE | `/carreras` | CRUD carreras + especialidades |
| POST | `/backup` | Disparar backup manual |
| GET | `/dashboard/stats` | Contadores: residentes activos, cerrados, empresas |

**Convención de respuesta**:
```json
// Éxito:  { "ok": true, ...datos }   HTTP 200/201
// Error:  { "error": "mensaje" }     HTTP 4xx/5xx
```

## 6. Frontend — Módulos (App.jsx)

Todo el UI vive en `proyecto/src/App.jsx` (~3500 líneas).  
Navegación por estado `activePage` (sin React Router).

| Módulo (`activePage`) | Componente principal | Descripción |
|---|---|---|
| `dashboard` | `Dashboard` | Stats + residentes recientes |
| `residentes` | `Residentes` | Tabla + modal CRUD + subida de docs |
| `proyectos` | `Proyectos` | Tabla de proyectos con hitos (es_expediente=0) |
| `empresas` | `Empresas` | CRUD empresas |
| `asesores` | `Asesores` | CRUD asesores internos/externos |
| `carreras` | `Carreras` | CRUD carreras + especialidades |

**Sistema de notificaciones** (custom, sin librería):
- `useNotification()` → `{ notify, alert, confirm }`
- `notify(msg, type)` → Toast glassmorphism auto-dismiss (3.2s)
- `alert(msg)` → Dialog bloqueante con OK
- `confirm(msg)` → Dialog bloqueante con Cancelar/Confirmar

**Tema dark mode**: toggle en sidebar → agrega clase `dark` a `document.body`.

## 8. Setup rápido (nuevo colaborador)

```bash
# 1. Clonar
git clone https://github.com/Hyperys/SoftwareResidenciasITCM.git
cd SoftwareResidenciasITCM

# 2. Backend
python -m venv .venv
.venv\Scripts\activate
pip install -r backend/requirements.txt
cp backend/database.example.py backend/database.py
# → Editar database.py con tus credenciales MySQL

# 3. Crear la BD
mysql -u root -p < database/schema.sql

# 4. Frontend
cd proyecto
npm install
npm run dev        # http://localhost:5173

# 5. Backend (terminal separada)
cd backend
python app.py      # http://localhost:5000
```

## 9. Decisiones de diseño notables

| Decisión | Razón |
|---|---|
| App.jsx monolítico | Historia del proyecto; refactoring pendiente en features futuras |
| CSS-en-JS (template string) | Sin dependencias de styling; fácil de mantener para el equipo actual |
| Flask sirve la SPA | Simplifica el empaquetado con PyInstaller en un solo proceso |
| `es_expediente` flag | Separa "proyectos en trámite" de "expedientes formalizados" |
| PyInstaller + electron-builder | Distribución como .exe sin instalar Python/Node en cliente |
